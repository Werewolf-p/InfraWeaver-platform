InfraWeaver is primarily an operator console, but many day-to-day tasks can still be done from a phone or tablet when you are on-call or away from a workstation.

## What works well on mobile

Mobile is good for:

- checking cluster or service health
- reading logs and recent alerts
- starting or stopping a game server
- confirming DNS, status, or uptime values
- using the Wiki for operational instructions

## Navigation on small screens

The main dashboard swaps the persistent desktop sidebar for mobile-friendly navigation patterns. In the wiki specifically, the page tree opens as a bottom sheet so you can browse sections without losing the current article.

## Best practices

- pin the most-used pages so they are easier to reach
- keep VPN connectivity active before opening internal-only routes
- use mobile for quick actions, not bulk maintenance

## Editing from mobile

Simple actions such as toggling a service or checking a file are fine. Complex edits such as bulk manifest rewrites, large uploads, or multi-file troubleshooting are still much safer on desktop.

## Mobile caveats

- long code blocks are horizontally scrollable
- tables are condensed and may require sideways scrolling
- live log streaming is harder to follow on a narrow viewport
- clipboard-heavy workflows are slower than on desktop

> **Note:** If you are troubleshooting an incident, mobile is best used for triage and acknowledgement. Switch to a desktop session for deeper fixes.
